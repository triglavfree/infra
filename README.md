# Infra

Минималистичный и переносимый infrastructure-проект для запуска Forgejo и Woodpecker
в контейнерах с нулевым управлением хост-системой.

Проект ориентирован на простоту, идемпотентность и жёсткие архитектурные границы.

---

## Design principles

### Единственный источник истины — Git

Вся инфраструктура описывается **исключительно кодом в этом репозитории**.  
Хост-система не является объектом конфигурации и не управляется проектом.

Изменения в `/etc`, systemd-юнитах, пользователях и пакетах **не отслеживаются**.

---

### Контейнеры вместо конфигурации хоста

Все сервисы запускаются **только в контейнерах**:

- конфигурация передаётся при запуске контейнера
- secrets прокидываются через `sops + age`
- контейнеры считаются disposable и могут быть пересозданы в любой момент

Хост выполняет роль платформы, а не части инфраструктуры.

---

### Минимальный контракт с хостом

От сервера требуется только:

- `podman`
- `sops`
- `age`
- POSIX-shell
- доступ к сети и диску

`systemd`, `runit`, `openrc` **не используются и не требуются**.

---

### Secrets

Secrets:

- хранятся в Git **только в зашифрованном виде** (`*.sops`)
- расшифровываются **только в runtime**
- никогда не пишутся в plaintext на диск

Git содержит форму секрета, но не его содержимое.

---

### Переносимость важнее оптимизации

Архитектура ориентирована на сценарий:

```bash
git clone <repo>
cd infra
./bootstrap.sh --install
infra up all
```
Независимо от дистрибутива (Ubuntu, Debian, Alpine, Void).

### Структура проекта

```txt
infra/
├── bin/          # CLI (infra)
├── scripts/      # Проверки и запуск контейнеров
├── services/     # Описание сервисов
│   ├── forgejo/
│   └── woodpecker/
├── secrets/      # Helpers для sops / age
├── bootstrap.sh  # Подготовка системы
└── README.md
```
---

### Frozen stack

Проект использует минимальный и осознанно зафиксированный набор компонентов:
```txt
✔ Forgejo    — Git, issues, source of truth
✔ Woodpecker — CI/CD для Forgejo
```
---

### Bootstrap

`bootstrap.sh` — одноразовый скрипт подготовки системы перед использованием infra.

```bash
./bootstrap.sh --install
```

### Использование infra

* Запуск всех сервисов:
```bash
infra up all
```
* Запуск одного сервиса:
```
infra up forgejo
```
* Остановка одного сервиса:
```
infra down forgejo
```
* Статус контейнеров:
```
infra status
```

### Мониторинг

podman stats
